## AGENTS



Proyecto: Portafolio personal



Rol del agente: Desarrollador web experto con 12 años de experiencia.



Objetivo: Crear una aplicación web para un portafolio personal Julian Arango

donde podamos tener nuestros proyectos, habilidades y experiencia laboral como

AI engineer / Data Analyst.



Dato importante: Todos los datos, proyectos, habilidades, experiencia laboral deben ser inventados pero que puedan ser creados mas adelante (los debe crear y añadir el agente de ia)





Funcionalidades de la aplicación:



- Parte publica:

- Pagina home

- Seccion experiencia

- descripcion general de mi y mi experiencia laboral

- Seccion habilidades

- lenguajes que usamos

- tipos de proyectos que hemos realizado

- frameworks herramientas

- barras con las habilidades (AI engineer, data analyst, desarrollo con ia, et)

- Seccion listado de proyectos

- proyectos con su nombre, descripcion, tecnologias usadas, enlace al proyecto, imagen.

- Seccion sobre mi

- descripcion general de mi a nivel personal

- seccion formulario de contacto

- Navegacion entre secciones

- Optimizacion seo basica





- Parte privada:

- Panel de administración de proyectos:

- Listado de proyectos

- Formulario de creacion de proyectos

- Edicion de proyectos

- Eliminacion de proyectos

- Enlazar a GitHub



- Backend:

- Haremos el backend en Supabase

- Login y registro de usuarios con Supabase

- Y se guardarán los proyectos en una base de datos de Supabase



Stack de tecnología:

- HTML5

- CSS3

- TailwindCSS

- JavaScript

- React

- Supabase



Preferencias generales importantes:

- Todos los textos visibles en la aplicación web debe estar en Español.



Preferencias de diseño:

- Basate en las imágenes del diseño y en el HTML del diseño que tienes en la carpeta design del proyecto.



Preferencias de estilos:

- Seguir a partir de los estilos de Tailwind que tiene ya el proyecto

- Colores (los del diseño)

- Que la webapp sea responsive





Preferencias de código:

- No añadas dependencias externas

- HTML debe ser semantico

- No uses alert, confirm, prompt, todo el feedback debe ser visual en el dom

- Prioriza el código legible y mantenible

- Prioriza que el codigo sea sencillo de entender

- Si el agente duda, que revise las especificaciones del proyecto y si no que pregunte al usuario





Estructura de archivos:

- la estructura de archivos del framework que estamos usando.
